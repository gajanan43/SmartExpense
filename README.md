# 💸 SmartExpense Tracker

A full-stack expense tracking web application built with **Spring Boot** REST API backend and **Vanilla JS** frontend. Supports secure user authentication, per-user data isolation, and is fully deployed on the cloud.

🔗 **Live Demo:* [https://smartexpense-frontend.onrender.com](https://smartexpense1.onrender.com/)

## ✨ Features

- 🔐 **JWT Authentication** — Register, login, and secure stateless sessions
- 👤 **Per-user data isolation** — Each user sees only their own expenses
- 📊 **Dashboard** — Total spent, monthly summary, top category, and category chart
- ➕ **Add Expense** — Title, amount, category, date, and notes
- 📋 **History** — Filter expenses by category and month
- 🗑️ **Delete** — Remove individual expense records
- ☁️ **Cloud deployed** — Backend on Render, MySQL on Aiven

## 🚀 API Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/register` | ❌ Public | Register new user |
| POST | `/api/login` | ❌ Public | Login and get JWT token |
| GET | `/api/expenses` | ✅ JWT | Get all expenses for logged-in user |
| GET | `/api/expenses/{id}` | ✅ JWT | Get expense by ID |
| POST | `/api/expenses` | ✅ JWT | Create new expense |
| PUT | `/api/expenses/{id}` | ✅ JWT | Update expense |
| DELETE | `/api/expenses/{id}` | ✅ JWT | Delete expense |
| GET | `/api/analytics/categories` | ✅ JWT | Get category-wise totals |

## ⚙️ How JWT Auth Works

```
User registers → password BCrypt encrypted → saved to DB
      ↓
User logs in → credentials verified → JWT token generated
      ↓
Frontend stores token in localStorage
      ↓
Every API request sends: Authorization: Bearer <token>
      ↓
JwtFilter validates token → extracts username
      ↓
Controller fetches only that user's data ✅
```

## 🏃 Run Locally

### Prerequisites
- Java 21+
- Maven
- MySQL
- Node.js (optional, for frontend)

### Backend Setup

**1. Clone the repo**
```bash
git clone https://github.com/gajanan43/SmartExpense.git
cd SmartExpense/backend
```

**2. Configure `application.properties`**
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/smartexpense
spring.datasource.username=root
spring.datasource.password=yourpassword
spring.jpa.hibernate.ddl-auto=update
jwt.secret=your_base64_secret_key
server.port=8080
```

**3. Run the backend**
```bash
mvn spring-boot:run
```
Backend starts at → `http://localhost:8080`

### Frontend Setup

**1. Create `config.js`** in the frontend folder:
```js
window.API_BASE = 'http://localhost:8080/api';
```

**2. Open `index.html`** with VS Code Live Server

Frontend runs at → `http://127.0.0.1:5500`

## 🐳 Run with Docker

```bash
cd backend

# Build image
docker build -t smartexpense-backend:1.0.0 .

# Run container
docker run -p 8080:8080 \
  -e DB_URL=jdbc:mysql://your-db-host:3306/defaultdb \
  -e DB_USERNAME=your_username \
  -e DB_PASSWORD=your_password \
  -e JWT_SECRET=your_secret \
  smartexpense-backend:1.0.0
```

## ☁️ Deployment

| Service | Platform | URL |
|---------|----------|-----|
| Frontend | Render Static Site | `smartexpense-frontend.onrender.com` |
| Backend | Render Web Service (Docker) | `smartexpense-backend-latest.onrender.com` |
| Database | Aiven Cloud MySQL | — |

### Render Environment Variables (Backend)

| Key | Value |
|-----|-------|
| `DB_URL` | `jdbc:mysql://HOST:PORT/defaultdb?useSSL=true&serverTimezone=UTC` |
| `DB_USERNAME` | your aiven username |
| `DB_PASSWORD` | your aiven password |
| `JWT_SECRET` | your base64 secret key |

### Render Environment Variables (Frontend)

| Key | Value |
|-----|-------|
| `API_BASE` | `https://smartexpense-backend-latest.onrender.com/api` |
